# Satara RTO Research Pack

Research date: 2026-04-06
Region: Satara district, Maharashtra, India

This folder stores the Satara RTO research in category-based files so the information can be reused for product planning, content writing, checklist generation, and workflow design.

## Folder structure

- `official/office-and-jurisdiction.md`
  - Official office details for Satara district and portal routing notes.
- `official/licence-services.md`
  - Driving licence related services: learner's licence, permanent licence, renewal, duplicate, address change, IDP.
- `official/vehicle-services.md`
  - Vehicle registration, ownership transfer, NOC, RC renewal, duplicate RC, address change, hypothecation, fitness.
- `official/permit-tax-puc.md`
  - Permit, tax, and PUC related research.
- `documents/document-checklists.md`
  - Reusable document bundles and service-wise checklist notes.
- `documents/practical-additional-documents.md`
  - Public-reported backup papers that users say are often still asked for during verification.
- `documents/forms-reference.md`
  - Forms reference for Sarathi, Vahan, registration, permit, and fitness workflows.
- `product/user-pain-points.md`
  - Publicly visible complaints, reviews, and common friction patterns.
- `product/website-feature-opportunities.md`
  - Product requirements and website opportunities derived from the research.
- `sources/official-sources.md`
  - Official sources only: Maharashtra Transport Department, Parivahan, Vahan, Sarathi.
- `sources/public-sources.md`
  - Public sources used only for user pain-point research.
- `data/service-matrix.csv`
  - Compact service matrix for implementation, search, filters, or checklist engines.

## Important notes

- Official process details should be treated as the primary source of truth.
- Public pain points are included to guide UX and support design, not to override official rules.
- Some fee values on state pages and live portals can differ because of user charges, smart-card fees, or portal-calculated extras. Use the live portal amount at payment time for final confirmation.
- Permit and tax services are highly vehicle-type dependent. The website should compute eligibility and fees dynamically instead of hardcoding one flat answer.
